GUIA DE SUSTENTACION ORAL - SUNARP SQL
======================================

Objetivo del discurso
---------------------
Explicar que el proyecto resuelve un problema de seguimiento manual de titulos
registrales mediante un sistema academico en Python con base de datos SQL real.

Orden sugerido de exposicion
----------------------------
1. Problema:
   "El seguimiento manual genera demora, errores de transcripcion y poca
   trazabilidad. Por eso se analizo con Ishikawa."

2. Regla clave:
   "En SUNARP se consulta por numero de titulo. La partida no se usa como
   criterio de busqueda SUNARP; se guarda en SQL para control interno."

3. Solucion:
   "Comparamos Excel/manual, Python con CSV y Python con SQL. Se eligio SQL
   porque permite persistencia, historial, reportes y revision por phpMyAdmin."

4. Codigo:
   "El sistema esta separado en capas: main.py para menu, repository.py para
   CRUD, database.py para conexion, importador CSV y demo de evidencia."

5. Base de datos:
   "La base tiene titulos, lotes e historial. Tambien vistas para resumen por
   estado y avance por bloque."

6. Pruebas:
   "Hay 10 casos documentados y 7 pruebas automaticas. Tambien se genera una
   evidencia tecnica con CRUD temporal que se limpia al final."

7. Demo recomendada:
   Abrir USO_SIMPLE.bat.
   Opcion 1: estado SQL.
   Opcion 7: pruebas automaticas.
   Opcion 8: evidencia tecnica.
   Opcion 5: menu CRUD.

Frases para preguntas dificiles
-------------------------------
- Si preguntan por SQL:
  "La persistencia se hace en MySQL/MariaDB con tablas, vistas e historial."

- Si preguntan por partida:
  "Partida es un dato registral de control dentro de la base; la consulta
  operativa se hace por titulo."

- Si preguntan por escalabilidad:
  "La arquitectura permite importar CSV, trabajar por lotes y generar reportes
  SQL sin cambiar el menu principal."

- Si preguntan por errores:
  "El programa valida entradas y reintenta sin cerrarse. Ademas, las pruebas
  automaticas cubren busqueda, historial, exportacion, importacion y partida."

Cierre sugerido
---------------
"El resultado no solo ejecuta un menu, sino que deja datos persistentes, evidencia
tecnica, trazabilidad y pruebas. Eso permite sustentar el problema, la solucion
y el codigo con documentos verificables."
