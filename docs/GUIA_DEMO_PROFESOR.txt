GUIA RAPIDA DE DEMOSTRACION PARA EL PROFESOR
============================================

Objetivo
--------
Demostrar que el proyecto usa SQL real, tiene CRUD, conserva historial y
distingue correctamente entre:

- titulo: dato usado para consulta SUNARP.
- partida: dato guardado en SQL para control interno.

Secuencia recomendada
---------------------
1. Abrir USO_SIMPLE.bat.
2. Opcion 1: verificar MySQL y resumen de base.
3. Opcion 4: crear/actualizar base demo si el profesor pide reiniciar.
4. Opcion 7: ejecutar pruebas automaticas.
5. Opcion 8: generar evidencia tecnica.
6. Opcion 5: abrir menu CRUD si el profesor quiere operar el sistema.
7. Abrir la carpeta 06_casos_demo_por_estado y usar la guia para mostrar
   un ejemplo INSCRITO, OBSERVADO, TACHADO, PENDIENTE, CALIFICACION y
   NO_ENCONTRADO.

Prueba tecnica esperada
-----------------------
El archivo salidas/evidencia_demo_profesor.txt debe mostrar:

- resumen general de registros;
- resumen por estado;
- busqueda por titulo 393004;
- validacion interna por partida P03010001;
- prueba CRUD temporal;
- historial de cambio de estado;
- pendientes por bloque.

Casos para demostrar clasificacion
----------------------------------
La carpeta 06_casos_demo_por_estado contiene titulos reales del dataset
separados por estado. Use estos titulos para demostrar que el sistema no solo
consulta datos, sino que tambien clasifica correctamente cada situacion del
tramite.

Punto clave para sustentar
-------------------------
SUNARP se consulta por numero de titulo. La partida se guarda e indexa en SQL
como evidencia y control interno, pero no se presenta como criterio de busqueda
en SUNARP.
