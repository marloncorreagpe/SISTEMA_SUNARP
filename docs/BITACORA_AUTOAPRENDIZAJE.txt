Priorizacion de brechas
-----------------------
CRITICAS (bloquean el objetivo del proyecto):
  1. Persistencia real de datos (sin SQL no hay seguimiento confiable).
  2. Diferenciar titulo y partida (es la regla del dominio; un error aqui invalida la consulta).
  3. Validacion robusta (sin ella el programa se cae con datos reales).
SECUNDARIAS (mejoran calidad y usabilidad, no bloquean):
  4. Evidencia reproducible.
  5. Pruebas automatizadas.
  6. Formularios graficos e interfaz de usuario.

Antes / Despues
---------------
Antes: el flujo dependia de ejecutar comandos separados y recordar rutas.
Despues: USO_SIMPLE.bat centraliza preparacion, pruebas, importacion y evidencia.

Antes: la partida podia confundirse como criterio de consulta.
Despues: el menu indica "Validar partida en SQL (control interno)" y la busqueda SUNARP queda por titulo.

Antes: el historial podia estar vacio en una base recien cargada.
Despues: el seed demo deja una evidencia inicial de trazabilidad.

Antes: el sistema solo se operaba por consola.
Despues: 3 formularios graficos permiten registrar, consultar (con estado por color) y actualizar.
